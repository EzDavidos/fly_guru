Документ-основа проекта. Используется как контекст для разработки с Claude Code / Opus. Спорные места помечены **[TBD]** — подставить значения после утверждения с руководителем.

## 1. Обзор проекта

Сайт-экосистема школы электрофойлов в Нячанге (Вьетнам):
- Обучение катанию (запись через формы, без онлайн-оплаты)
- Клубная система (абонементы, членство, сафари, путешествия)
- Витрина продажи фойлов (основная аудитория покупателей — РФ)
- Реферальная система (агенты + «приведи друга» для членов клуба)
- CRM для админа и инструкторов

Язык на запуске: русский. Архитектура закладывается под i18n (en, vi — позже).
Оплаты онлайн нет: формы заявок, оплата на месте (QR/крипта — вне сайта).

## 2. Механика клуба

### Путь клиента
1. **Тандем** — пробный полёт с инструктором. Без обязательств, членства не даёт.
2. **Базовое обучение** — входная точка. После него допуск к самостоятельному прокату, сафари, путешествиям.
3. **Абонемент 300 мин** — можно купить в любой момент. Если клиент ещё не обучен, первые 60 минут абонемента = обучающее занятие (обучение обязательно).
4. **Членство клуба** — активируется покупкой первого абонемента. Статус бессрочный.

### Условия членства
- Ответ руководителя (2026-07): членство даётся после прохождения базового обучения (а не после первого абонемента) и активно постоянно. В CRM membership создаётся при продаже абонемента.
- Выезды обученным без абонемента — **РЕШЕНО (пак G, 2026-07-15):** экскурсия/сафари доступны по одобрению инструктора (флаг `clients.tour_approved`), абонемент для них не обязателен. Что остаётся эксклюзивом абонемента/клуба — клубный чат, мероприятия, «приведи друга», скидки (см. привилегии члена клуба).
- Срок жизни минут абонемента: **3 месяца**. Неиспользованный остаток можно передать другу **[TODO: механика передачи — вручную через админа/инструктора или из кабинета клиента; решить к этапу 4–5]**.
- В БД сразу заложить поле `level` у члена клуба (Member/Rider/Legend) — интерфейс уровней делается второй волной, не на запуске.

### Реферальная система
Единая механика, два типа рефереров:

| | Агент | Член клуба («приведи друга») |
|---|---|---|
| Ссылка | уникальная, создаёт админ | уникальная, генерится автоматически в кабинете |
| Скидка другу/клиенту | 200 000 ₫ на базовое обучение | 200 000 ₫ на базовое обучение |
| Награда рефереру | 300 000 ₫ (деньгами) | +10 мин за обучение друга, +30 мин если друг купил абонемент; за тандем — ничего |
| Условие начисления | клиент прошёл базовое занятие | друг прошёл базовое занятие / купил абонемент |

## 4. Структура сайта (публичная часть)

```
/                Главная: оффер, видео, путь клиента, FAQ (высоко, 2–3 блоком),
                 блок отзывов (2–3 лучших + ссылка на /reviews),
                 тизер магазина, CTA «Записаться»
/training        Обучение: базовые (взрослый/детское/вдвоём), сжатый FAQ, форма записи
/tandem          Тандем: «попробуй за 5 минут», цены, форма
/club            Клуб: абонемент, членство, экскурсия, путешествия, сафари,
                 приведи друга (всё members-only, кроме покупки абонемента)
/shop            Витрина фойлов (каталог)
/shop/[model]    Карточка фойла + форма «узнать цену / заказать»
/prices          Сводный прайс
/reviews         Отзывы (все)
/contacts        Контакты, карта, соцсети
/r/[code]        Реф-лендинг: продающий блок + скидка 200к + форма с зашитым кодом
/login           Вход (все роли)
```

Что НЕ выносится в отдельные страницы:
- Экскурсия, путешествия, сафари, прокат — блоки внутри `/club`
- Видеосъёмка — упоминается в `/prices` и в релевантных блоках
- FAQ — блок на главной (высоко) + сжатая версия на `/training`, не отдельная страница
- Виды обучения — блоки внутри `/training`, не три страницы

Ключевые моменты:
- Магазин — отдельная страница; на главной только небольшой блок-переходник.
- Отзывы: отдельная страница `/reviews` + блок с 2–3 лучшими на главной.
- Реф-лендинг `/r/[code]` — отдельная страница, не query-параметр на главной. Обязательный контент: «90% едут уже на первом занятии», комфортные доски, связь с инструктором на воде, опыт инструкторов, явная скидка 200к.
- Витрина: аудитория из РФ → следить за скоростью загрузки, оптимизация изображений обязательна.

## 5. Роли и кабинеты

### Роли: `admin`, `instructor`, `member`, `agent`

**Кабинет админа (`/admin`):**
- Все заявки (статусы: новая → подтверждена → проведена / отменена)
- Все сессии; возможность добавить/исправить запись задним числом на любую дату и любого инструктора (кейс: инструктор записал на бумагу)
- Дашборд: выручка, популярность услуг, динамика
- Месячный расчёт: по инструкторам (клиенты, выручка, ЗП 10%), по агентам (приведённые, комиссия), общая маржа
- Управление: агенты (создание рефкодов), услуги/цены, инструкторы, члены клуба (создание + инвайт-ссылка), абонементы (продажа/активация)

**Кабинет инструктора (mobile-first):**
- Заявки на сегодня/ближайшие
- Записать клиента: имя, телефон, услуга → сессия атрибутируется этому инструктору
- Списать минуты с абонемента: поиск клиента → виден остаток → внести каталку (минуты)
- Моя статистика за месяц: клиенты, выручка, моя ЗП

**Кабинет члена клуба:**
- Остаток минут (крупно, живое число)
- История каталок
- Своя ссылка «приведи друга» + статистика начисленных минут
- Сафари/путешествия: инфо + запись

**Кабинет агента:**
- Своя реф-ссылка
- Статистика: переходы → заявки → подтверждённые клиенты
- Начисленная комиссия за текущий/прошлый месяц

### Онбординг аккаунтов (онлайн-оплаты нет)
- Член клуба: купил абонемент оффлайн → админ создаёт клиента + абонемент в CRM → система генерит инвайт-ссылку → отправляется клиенту в мессенджер → клиент задаёт пароль.
- Агент: создаёт только админ.
- Логин (утверждено): **email или телефон + пароль**, без SMS. Основной вариант — настоящий email (позволяет автоматически создать аккаунт). Если email нет — при создании аккаунта генерится технический email из телефона, вход по телефону работает через резолв телефон → email на сервере.

## 6. Модель данных (Supabase / Postgres)

```
users            id, role (admin|instructor|member|agent), name, phone, email, auth_id
agents           id, user_id, ref_code, commission_fixed (300000), active
clients          id, name, phone, source (agent|member|site|offline),
                 referrer_type, referrer_id, created_by (instructor/admin id)
services         id, name, duration_min, price, category
                 (training|tandem|rental|tour|subscription|extra), active
bookings         id, client_name, phone, service_id, preferred_date,
                 ref_code (nullable), status (new|confirmed|done|cancelled),
                 created_at, client_id (nullable, после сопоставления)
sessions         id, client_id, service_id, instructor_id, date,
                 minutes_used (nullable — для списаний с абонемента),
                 amount (чек; 0 для списаний с абонемента),
                 subscription_id (nullable), created_by, created_at
subscriptions    id, client_id, total_minutes (300), price (6000000),
                 sold_by (instructor_id), sold_at, expires_at (+3 мес от продажи),
                 status (active|expired|used_up)
                 -- остаток = total_minutes − SUM(sessions.minutes_used)
referral_rewards id, referrer_type (agent|member), referrer_id, client_id,
                 reward_type (money|minutes), amount, status (pending|confirmed),
                 confirmed_at
memberships      id, client_id, level (member|rider|legend), since,
                 -- активность привилегий вычисляется по наличию активного абонемента
products         id (магазин), name, brand, specs (jsonb), price, photos, active
product_requests id, product_id, name, phone/contact, message, created_at
```

Row-Level Security (Supabase RLS):
- instructor: видит свои сессии/статистику, может создавать sessions и clients, видит остатки абонементов всех клиентов (нужно для списания)
- member: видит только своё (абонемент, сессии, рефералы)
- agent: видит только свою статистику
- admin: всё

## 7. Ключевая бизнес-логика

### ЗП инструктора
- 10% от чека каждой сессии, атрибутированной инструктору. Атрибуция = **кто записал, того и клиент** (записи с личного аккаунта инструктора).
- **Абонемент: 10% от 6 млн начисляется инструктору в момент продажи**, независимо от того, откатает ли клиент минуты. Последующие списания минут (`sessions` с `subscription_id`) идут с `amount = 0` и комиссию не генерируют.
- Админ может добавить сессию задним числом на любого инструктора (сверка с бумажными записями) — она участвует в расчёте ЗП как обычная.
- Месячный отчёт: `SUM(amount) * 0.10` по инструктору + 10% от проданных им абонементов.

### Абонемент
- Продажа: админ или инструктор создаёт `subscription` → клиент становится членом клуба (создаётся `membership`, если нет).
- Первый абонемент необученного клиента: первое списание = обучающее занятие, **60 мин** (утверждено).
- Списание: инструктор находит клиента → видит остаток → вносит минуты. Остаток не может уйти в минус (валидация; каталка больше остатка → доплата отдельной сессией по прайсу проката).
- Экспирация: `expires_at` = продажа + **3 месяца**, после — статус expired, минуты сгорают. Остаток можно передать другу **[TODO: механика передачи]**.

### Рефералы
- Переход по `/r/[code]` → форма записи с зашитым кодом → `booking.ref_code`.
- Когда заявка доведена до статуса `done` (базовое занятие проведено) → создаётся `referral_reward`: агенту money 300 000, члену minutes **+10 за обучение / +30 за купленный другом абонемент** (зачисляются на его активный абонемент; за тандем награды нет).
- Скидка 200 000 применяется к чеку сессии (amount = 1 800 000 для взрослого базового) — комиссия инструктора считается от фактического чека.

### Заявки с сайта
- Форма: имя, телефон/мессенджер, услуга, желаемая дата, комментарий.
- Попадают в CRM со статусом `new`. Уведомление админу в Telegram — сделано (этап 2, src/lib/telegram.ts).

## 10. Этапы разработки (порядок промптов для Opus)

- **Этап 0.** Сетап: Next.js + Supabase, репозиторий, деплой на Vercel, структура проекта, схема БД (миграции), сиды услуг
- **Этап 1.** Публичные страницы со статичным контентом (главная, обучение, тандем, клуб, прайс, контакты) + дизайн-система
- **Этап 2.** Формы записи → запись в bookings + реф-лендинг `/r/[code]` + уведомления
- **Этап 3.** Auth + роли + кабинет инструктора (запись клиентов, списание минут, своя статистика)
- **Этап 4.** Админка: заявки, сессии, правки задним числом, абонементы, дашборд, месячные расчёты (ЗП, комиссии)
- **Этап 5.** Кабинеты члена клуба и агента + механика «приведи друга»
- **Этап 6.** Магазин: каталог, карточки, заявки на покупку
- **Этап 7.** Полировка: SEO, оптимизация под РФ, i18n-каркас

Правило работы: один этап = один изолированный блок изменений. Перед переходом к следующему — проверка чек-листа «готово» этапа. Не смешивать этапы в одном промпте.
